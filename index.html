Platform overview

This HTML is a front-end prototype. Server-side endpoints, database schemas and broker API integration should be implemented in your backend (see comments in the Strategy Builder and Broker sections).

1. Backend / Server

Runs bots, schedules tasks, exposes REST/WebSocket endpoints for the UI and execution engine.

2. Database

Store users, strategies, trades and logs (SQL or NoSQL). Support migrations & backups.

3. Strategy Builder

Visual UI to assemble strategy rules, indicators, and risk settings (save/load JSON).

4. Execution Engine

Scheduler/worker that evaluates strategies and sends orders to broker APIs.

5. Broker Integration

Plugin-able adapters (Deriv, Binance, etc.) that translate strategy actions to real orders.

6. Auth & Accounts

OAuth / JWT authentication and role-based access (user, admin, dev).

Bot Strategy Builder

Create or edit strategy logic. Save as JSON to your backend. The "Run" button will send the strategy to the execution engine (via API).

Backend endpoints (suggestions):
  • POST /api/strategies — save strategy JSON
  • GET /api/strategies/:id — fetch strategy
  • POST /api/execute/:strategyId — ask execution engine to run once

Execution Engine (Workers)

Engine responsibilities:

  • Poll market data (or subscribe via streaming)
  • Evaluate enabled strategies
  • Place/cancel orders via broker adapters
  • Log actions to database & push events via WebSocket to UI

Suggested components

Scheduler

Queue and run jobs reliably (e.g. Celery / RQ / Sidekiq)

Worker

Isolated process for strategy evaluation

Streamer

Market data subscriber (WebSocket)

Broker / API Integration

Adapter pattern recommended. Example: Deriv, Binance, MetaTrader, FIX.

  • Store API keys encrypted server-side
  • Permission scopes (trade/read)
  • Testnet & live mode toggle

User Accounts & Authentication

Support signup/login, email verification, password reset and session management.

  • Use JWT + refresh tokens or server sessions
  • 2FA for trading actions
  • Role-based access: user, admin

Dashboard / Analytics

Visualize bot performance, drawdowns, trade history and balance over time.
